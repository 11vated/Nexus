The MetaMind Framework: Theory of Mind for Multi-Agent Systems

Through a self-reflective, bidirectional inference loop, each agent develops a metacognitive ability in a self-supervised manner—and then generalizes this ability from first-person to third-person through analogical reasoning.

The ALMA Architecture: Knowledge Stratification for Adaptive Systems

Pillar 1: Meta-Cognitive Agent Loop
Instead of executing tasks blindly, the agent should reason about its own reasoning. The Co-Regulation Design Agentic Loop (CRDAL) introduces a metacognitive co-regulation agent that assists the design agent in metacognition to mitigate design fixation.

Pillar 2: Stratified Knowledge Architecture
Replace the flat vector database with a multi-layer knowledge stratification system similar to ALMA. Organize code knowledge into layers:

Layer 0: Syntax & Types — Concrete code tokens, type definitions, function signatures

Layer 1: Control Flow & Data Flow — How functions call each other, data dependencies

Layer 2: Design Patterns & Architectural Styles — Repository pattern, factory pattern, event sourcing

Layer 3: Domain Concepts & Business Logic — "Payment authorization" vs "refund processing"

Layer 4: Developer Intent & Design Philosophy — "This system prioritizes consistency over performance"

The "membranes" between layers are selectively permeable — the Architect agent operates primarily at Layers 2–4, the Developer at Layers 0–2, but insights can flow upward and downward.

The benchmark Design Constraint Benchmark reveals that fewer than half of resolved issues are fully design‑satisfying, and design violations are widespread, motivating design‑aware evaluation beyond functional correctness.

Pillar 4: Underspecification Handling & Interactive Clarification
The Ambig-SWE benchmark shows that current models struggle to distinguish between well-specified and underspecified instructions. Pillar 5: Multi-Memory Architecture with Cross-Session Cognitive Collaboration
